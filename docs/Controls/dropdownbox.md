## 一、概述

当下拉框选项被选中时，可触发数据更新、画面跳转等交互操作。支持手动配置选项列表，实现灵活的业务场景适配

![图片描述](../images/控件照片/下拉框/1.png)

## 二、使用场景

下拉框控件在以下场景中使用：

### 1.数据选择场景

- 当需要用户根据不同的选项下发不同的命令时
- 需要提供标准化输入，避免用户自由输入错误数据

### 2.功能触发场景

- 根据选择更新其他控件的数据显示
- 实现画面跳转或弹窗显示切换

## 三、快速开始

1. 拖拽下拉框控件到画布并调整位置尺寸
2. 在"选项内容"中配置选项列表（文本和值）
3. 设置"选中文本"和"选中值"确定默认选项
4. 配置样式属性（颜色、边框、字体等）（**可选**）
5. 设置"动作"定义选择后的交互行为（**可选**）

## 四、特殊事件

| 值变化   |                                                                                                                                                                                                                                                                                                                                    |                                                                      |
| -------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------- |
| 说明     | 当下拉框的选中值发生改变时， 对应的事件将会被触发                                                                                                                                                                                                                                                                                  |                                                                      |
| 入参     | 选中值（选中文本）                                                                                                                                                                                                                                                                                                                 |                                                                      |
| 操作类型 | 导航                                                                                                                                                                                                                                                                                                                               | 作时，控件选项的“值”必须与目标画面的名称完全一致，否则无法正确跳转。 |
|          | 变量写值                                                                                                                                                                                                                                                                                                                           | 会将选中的值写入到变量中                                             |
| 配置流程 | 1.在控件“动作”设置中，选择“值变化”作为触发事件<br />2.选择“导航”作为执行动作类型<br />3.设置目标画面，并确保选项值与画面名称一致<br />4.保存配置，完成自动化跳转设置<br />通过配置“值变化”事件，可以让控件在用户选择后自动执行后续流程，极大地提升了界面的自动化程度与用户体验。![图片描述](../images/控件照片/下拉框/4.gif)图 1-3 |                                                                      |
