## 一、概述

DipuCore 面向系统管理员提供菜单管理功能，用于集中管理和配置系统的导航菜单结构。它支持**多级菜单嵌套、**丰富的**菜单属性配置**，是构建清晰、安全、可维护的系统导航体系的核心工具。

## 二、界面与功能详解

### 1. 搜索筛选区

为了快速定位和管理海量菜单项，顶部提供了强大的搜索筛选功能：

- **按菜单名称搜索**：输入名称关键词，快速查找目标菜单。
- **按路由地址搜索**：通过页面的路由路径（URL）精准查找对应的菜单项。
- **按权限标识搜索**：输入权限编码，可精确查找绑定了特定权限的菜单，便于进行权限审计。

### 2. 菜单列表区

列表清晰展示了所有菜单项的配置详情，主要字段如下：

| 字段名称 | 描述与作用                                                                                                           |
| -------- | -------------------------------------------------------------------------------------------------------------------- |
| 名称     | 菜单项在导航栏中显示的文本标题。                                                                                     |
| 地址     | 该菜单项对应的页面路由路径（URL），点击菜单时将跳转至此。                                                            |
| 排序     | 权重值，用于决定同级菜单在导航栏中的显示顺序，数值越小排列越靠前。                                                   |
| 可见     | 开关控制，决定此菜单项是否在导航栏中对用户可见。可用于临时隐藏或基于条件显示菜单。                                   |
| 类型     | 区分菜单的种类：•功能菜单：可直接点击跳转页面的菜单项。•分组目录：仅用于组织下级菜单的容器（文件夹），自身不可点击。 |
| 操作     | 对该菜单项提供的功能操作按钮，包括添加、修改、删除。                                                                 |

### 3. 菜单操作功能区

在菜单列表的每一行，均提供三个核心操作按钮：

| 操作 | 功能与流程说明                                                                                                             |
| ---- | -------------------------------------------------------------------------------------------------------------------------- |
| 添加 | 在当前选中的菜单项下，创建其下级菜单。点击后弹出“添加菜单”表单，如图 1-4所示，填写名称、地址、图标、排序等配置后保存即可。 |
| 修改 | 修改当前菜单项的各项配置参数。点击后弹出编辑表单，如图 1-5所示，可对菜单的所有属性进行调整。                               |
| 删除 | 永久移除不再需要的菜单项。注意：此操作不可逆，且如果删除一个分组目录，其下的所有子菜单将一并被删除，请谨慎操作。           |

通过以上功能，管理员可以像搭积木一样，灵活构建出层次分明、权限清晰的系统导航菜单，并实现动态的菜单维护与更新。
