## 一、概述

任务管理模块主要用于满足用户**定时或周期性自动导出报表**的需求，同时提供完整的**导出日志记录**功能，方便用户追踪任务执行状态、查看历史记录并下载已生成的报表数据。

## 二、如何新建定时任务并导出数据

### 1. 创建新任务

点击 **“新增”** 按钮，进入任务配置页面。

- **任务名称**：输入本次定时任务的名称。
- **启用状态**：通过 **“启用”** 开关控制该任务是否执行。
- **报表选择**：在任务配置中，选择需要定时导出的**目标报表（来自数据分析-报表设计）**。
- **数据时间范围**：针对所选报表，设置每次导出时涵盖的**数据时间范围**（如：导出昨日全天数据、导出最近一小时数据等）。

![图片描述](../images/ManagementPlatform/任务管理2.png)

### 2. 配置调度策略（何时执行）

在 **“调度配置”** 中，选择任务的执行方式，共有三种类型：

| 调度类型 | 描述与配置                                                                                                          |
| -------- | ------------------------------------------------------------------------------------------------------------------- |
| 实时     | 立即执行一次。配置保存后，任务将立刻运行并导出数据。                                                                |
| 定时     | 在设定的未来时间点执行一次。可配置具体的年、月、日、时、分、秒。                                                    |
| 周期     | 在设定的时间范围内周期性重复执行。可配置为按分钟、小时、日、周、月等周期循环执行。例如：每天凌晨2点导出前一日报表。 |

配置完成后保存，任务将按照设定的策略自动运行。

![图片描述](../images/ManagementPlatform/任务管理3.png)

## 三、任务日志与记录

该模块提供所有报表导出任务的执行日志，用于监控和审计。

### 菜单列表区（日志字段说明）

| 字段名称 | 描述                                                   |
| -------- | ------------------------------------------------------ |
| 名称     | 导出报表的任务名称。                                   |
| 类型     | 任务的类型（导出报表）。                               |
| 信息     | 任务的信息（定时、实时）。                             |
| 最新     | 任务的状态（成功、失败）                               |
| 下次执行 | 任务预定的下一次执行时间（对于周期任务）。             |
| 创建人   | 该任务的创建者。                                       |
| 创建时间 | 任务的创建时间。                                       |
| 描述     | 对该任务的补充说明或备注。                             |
| 启用     | 任务是否启用                                           |
| 操作     | 对任务的的修改，以及日志查看，数据下载，删除任务等功能 |

通过此日志列表，用户可以清晰掌握所有导出任务的执行情况，对于成功的任务，可直接下载已生成的报表文件；对于失败的任务，可查看错误信息以便排查问题。

通过下载功能，用户可以将查询结果持久化保存，用于离线分析、数据存档、报告提交或与其他系统进行数据交换。
